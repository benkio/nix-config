#+TITLE: nix-config

Personal Nix flake for macOS (nix-darwin + Home Manager) and NixOS.

| Host   | System flake | Home Manager flake   |
|--------+--------------+----------------------|
| macOS  | ~.macos~     | ~.benkio@macos~      |
| NixOS  | ~.nixos~     | ~.benkio@nixos~      |

* macOS installation

Install in this order: Nix → nix-darwin → Home Manager.

** 1. Install Nix

Use the official multi-user installer from the Nix project:

- [[https://nix.dev/install-nix.html][nix.dev — Install Nix]]
- [[https://nixos.org/manual/nix/stable/installation/installing-binary][Nix manual — Installing a binary distribution]]

#+begin_src shell
curl --proto '=https' --tlsv1.2 -L https://nixos.org/nix/install | sh
#+end_src

Open a new terminal and verify:

#+begin_src shell
nix --version
#+end_src

*** macOS troubleshooting

If the installer complains about existing ~*.backup-before-nix~ shell profile backups,
remove the stale backups (after copying them somewhere safe) and run the installer again.

If you see ~the build users group 'nixbld' has no members~, the ~nixbld~ group in Open
Directory is missing ~GroupMembership~ entries. See
[[https://github.com/NixOS/nix/issues/6078][NixOS/nix#6078]].

After a macOS upgrade, if ~_nixbld1~ users are missing, see
[[https://github.com/NixOS/nix/issues/10892][NixOS/nix#10892]].

If Nix downloads fail with an SSL error about ~ca-certificates.crt~, it may point at a
removed Nix store path from a previous install. A temporary workaround is:

#+begin_src shell
sudo ln -sf /etc/ssl/cert.pem /etc/ssl/certs/ca-certificates.crt
#+end_src

Before the first ~darwin-rebuild~, nix-darwin may refuse to activate if ~/etc~ files
were modified manually or by a partial install. Rename the conflicting files as
instructed, e.g.:

#+begin_src shell
sudo mv /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/ca-certificates.crt.before-nix-darwin
sudo mv /etc/bashrc /etc/bashrc.before-nix-darwin
#+end_src

Then retry the ~darwin-rebuild~ / ~nix run nix-darwin -- switch~ command.

** 2. Clone this repository

#+begin_src shell
git clone https://github.com/benkio/nix-config.git ~/nix-config
cd ~/nix-config
#+end_src

** 3. Install nix-darwin

Official project and installation guide:

- [[https://github.com/nix-darwin/nix-darwin][nix-darwin on GitHub]]

On a fresh Nix install, flakes may not be enabled in ~/etc/nix/nix.conf~ yet. Use
~nix run~ for the first switch. You must pass ~--flake~ with this repository's
configuration — ~darwin-rebuild~ alone is not enough:

#+begin_src shell
cd ~/nix-config
sudo -H nix --extra-experimental-features nix-command \
  --extra-experimental-features flakes \
  run github:nix-darwin/nix-darwin -- switch --flake .#macos
#+end_src

Do /not/ run ~nix run nix-darwin#darwin-rebuild -- switch~ without ~--flake .#macos~;
that produces ~file 'darwin' was not found in the Nix search path~.

After the first successful switch, ~darwin-rebuild~ is on your PATH and experimental
features are configured by this flake. Later updates can use:

#+begin_src shell
cd ~/nix-config
sudo -H darwin-rebuild switch --flake .#macos
#+end_src

If ~darwin-rebuild~ is not on PATH yet (before Home Manager sets up your shell), use:

#+begin_src shell
sudo -H /run/current-system/sw/bin/darwin-rebuild switch --flake .#macos
#+end_src

** 4. Install Home Manager

Official documentation:

- [[https://nix-community.github.io/home-manager/][Home Manager manual]]
- [[https://github.com/nix-community/home-manager][Home Manager on GitHub]]

#+begin_src shell
cd ~/nix-config
home-manager switch --flake .#benkio@macos
#+end_src

If ~home-manager~ is not available yet, bootstrap it with:

#+begin_src shell
nix --extra-experimental-features nix-command \
  --extra-experimental-features flakes \
  run github:nix-community/home-manager -- switch --flake .#benkio@macos
#+end_src

Log out and back in (or open a new terminal) so shell and PATH changes take effect.

** Updating macOS configuration

Once everything is installed, rebuild system and user configuration:

#+begin_src shell
cd ~/nix-config
nix flake update
sudo -H darwin-rebuild switch --flake .#macos
home-manager switch --flake .#benkio@macos
#+end_src

On macOS, the ~update~ shell alias (defined in Home Manager) runs the above flow and
garbage collection.

* NixOS installation

** Graphical install (recommended)

Install from a USB drive and choose the minimal image. Skip the manual partitioning
steps below unless you need them.

Once in the installer environment:

#+begin_src shell
sudo loadkeys -v dvorak                    # optional: dvorak for the session
nmcli device wifi connect <ssid> password <password>
nix-shell -p git micro ripgrep
git clone https://github.com/benkio/nix-config.git
cd nix-config
sudo nixos-rebuild build --flake .#nixos
sudo nixos-rebuild switch --flake .#nixos
#+end_src

** Home Manager on NixOS

- [[https://nix-community.github.io/home-manager/][Home Manager manual]]

#+begin_src shell
cd ~/nix-config
home-manager switch --flake .#benkio@nixos
#+end_src

** Updating NixOS configuration

Channels are not used; versions are pinned in ~flake.lock~.

#+begin_src shell
cd ~/nix-config
nix flake update
sudo nixos-rebuild switch --flake .#nixos
home-manager switch --flake .#benkio@nixos
#+end_src

On NixOS, the ~update~ shell alias runs the above flow and garbage collection.

** Manual partitioning (historical)

The commands below were used for a manual install long ago. They may be outdated;
prefer the graphical installer when possible.

#+begin_src
nix-env -i git
git clone https://github.com/benkio/nix-config.git
cd nix-config/

sudo su

# MBR
parted /dev/sda -- mklabel msdos
parted /dev/sda -- mkpart primary 1MiB -8GiB
parted /dev/sda -- mkpart primary linux-swap -8GiB 100%

# UEFI
parted /dev/sda -- mklabel gpt
parted /dev/sda -- mkpart primary 512MiB -8GiB
parted /dev/sda -- mkpart primary linux-swap -8GiB 100%
parted /dev/sda -- mkpart ESP fat32 1MiB 512MiB
parted /dev/sda -- set 3 esp on

mkfs.ext4 -L nixos /dev/sda1
mkswap -L swap /dev/sda2
swapon /dev/sda2
mkfs.fat -F 32 -n boot /dev/sda3        # UEFI only
mount /dev/disk/by-label/nixos /mnt
mkdir -p /mnt/boot                      # UEFI only
mount /dev/disk/by-label/boot /mnt/boot # UEFI only
nixos-generate-config --root /mnt
cp configuration.nix /mnt/etc/nixos/
nixos-install
reboot
#+end_src

Remember the root password and remove the installer image from the drive.

* Post installation

Some setup still has to be done manually.

** Setup GitHub SSH and GPG keys

Go into Bitwarden for the note about GitHub SSH/GPG keys. Repeat for every
account/key.

#+begin_src shell
mkdir ~/.ssh
cd ~/.ssh
touch id_rsa id_rsa.pub
# Copy keys from Bitwarden into id_rsa and id_rsa.pub
chmod 600 ~/.ssh/id_rsa ~/.ssh/id_rsa.pub
# Copy GPG key into private.key, then:
gpg --import private.key
#+end_src

** Finish / check Emacs configuration

If ~.emacs.d~ was provisioned from the emacs-config repo on a ~fetchgit~ branch, set up
SSH keys first, then:

#+begin_src shell
git pull origin main
git checkout -f main
git clean -fx
git branch -D fetchgit
# Run generation of treesitter grammars
#+end_src
